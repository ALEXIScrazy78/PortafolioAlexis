Semana 10: Autenticación con JWT en Aplicaciones Web

¿Qué hemos aprendido?


Implementación de un sistema de autenticación seguro utilizando JSON Web Tokens (JWT). Para proteger los datos sensibles, se encriptarán las contraseñas con bcrypt. Se establecerá la conexión entre el backend, desarrollado con Node.js y Express, y el frontend, construido con Next.js, a través de peticiones HTTP. Finalmente, se procederá al despliegue de una base de datos MySQL en la nube utilizando Clever Cloud, y los servicios de backend y frontend se desplegarán en Render y Vercel, respectivamente.

Objetivos de Aprendizaje

  • Implementar autenticación segura con JWT
  • Gestionar conexiones entre frontend, backend y base de datos
  • Aprender buenas prácticas de seguridad (encriptación, validación de tokens)
  • Desplegar proyectos en entornos cloud.

Tecnologías Utilizadas

  • Backend: Node.js Express, MySQL, Sequelize, JWT, bcrypt, cors, dotenv
  • Fronted: Next.js, React, Bootstrap, Axios
  • Despliegue: Clever Cloud (MySQL), Render (backend), Vercel (frontend)
  • Control de versiones: Git, GitHub

¿Que se hizo?

1. Composición del sitio web con JWT

  • Backend: Se configuró un servidor con Node.js, Express y Sequelize para interactuar con MySQL. Se crearon rutas protegidas con middleware de autenticación JWT y se implementaron operaciones CRUD para usuarios, incluyendo el login con generación de tokens.
  • Fronted: Se desarrollaron las interfaces con Next.js y Bootstrap. Se manejaron los estados para la autenticación y la redirección al dashboard tras validar el token, estableciendo la conexión con el backend mediante Axios.
  • Despliegue: Se uno base de datos en Clever Cloud. Y para el backend en Render y frontend en Vercel.

Ejercicios

Despliegue de la aplicación web mediante Tokens

  • Este ejercicio presenta una autenticación basada en tokens es un método de verificación de identidad que permite a los usuarios acceder a sistemas sin necesidad de proporcionar credenciales constantemente. Se basa en la emisión de un token, un conjunto de datos cifrados, que valida la identidad del usuario por un período determinado.

Conclusión

El proyecto demostró cómo integrar JWT para autenticación en aplicaciones web full-stack, asegurando rutas y datos sensibles. La combinación de encriptación, tokens y despliegue en la nube proporciona una base escalable y segura para sistemas de login.

Reflexión

La autenticación con JWT no solo mejora la seguridad, sino que también optimiza la experiencia del usuario al evitar reinicios de sesión innecesarios. Sin embargo, es crucial gestionar adecuadamente la expiración de tokens y proteger las claves secretas. Este proyecto reforzó la importancia de conectar teoría (conceptos de seguridad) con práctica (implementación y despliegue).
