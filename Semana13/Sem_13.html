Semana 13: Desarrollo Backend PHP y Laravel

¿Qué hemos aprendido?


Esta semana instalamos y configuramos Apache HTTP Server en modo portable, añadimos PHP como intérprete del lado del servidor y gestionamos dependencias con Composer. A continuación dimos el salto al framework Laravel, donde practicamos la creación de un proyecto desde cero, la conexión a MySQL, la definición de migraciones y modelos Eloquent, y el ajuste de la interfaz con Tailwind CSS. También exploramos la CLI Artisan, la estructura de carpetas y el ciclo de vida de una petición MVC en Laravel.

Objetivos de Aprendizaje

  • Comprender el flujo petición–respuesta en Apache + PHP
  • Instalar PHP manualmente y habilitar extensiones esenciales
  • Gestionar dependencias de proyecto con Composer
  • Construir un CRUD básico con Eloquent, controladores y vistas Blade
  • Probar endpoints y formularios desde Postman y el navegador

Tecnologías Utilizadas

  • Backend: PHP 8.x, Laravel 12 .x, Eloquent ORM
  • Servidor: Apache HTTP Server (Apache Lounge build)
  • Gestor de dependencias: Composer
  • Base de datos: MySQL
  • Frontend: lade + Tailwind CSS
  • IDE: VS Code o IntelliJ IDEA + Artisan, npm

¿Que se hizo?

1. Configuración del entorno y Composer

  • Se configuró Apache Lounge (puerto 8080), se instaló PHP manualmente y se habilitaron extensiones como mysqli.
  • Se añadió PHP al PATH y se instaló Composer para gestionar dependencias, verificando su funcionamiento con composer -V.

2. Creación del proyecto en Laravel y conexión a base de datos

  • Se generó un nuevo proyecto con laravel new estudiantes-app, se configuró el archivo .env para conectar con MySQL
  • Se crearon migraciones y modelos como Estudiante, que se aplicaron con php artisan migrate.

3. Desarrollo y prueba de la aplicación

  • Se diseñaron rutas (web.php), un controlador (EstudianteController) y una vista Blade con Tailwind CSS para registrar estudiantes.
  • Finalmente, se probó la funcionalidad desde el navegador y con Postman mediante php artisan serve.

4. Verificación con Postman

  • El testeo de este programa se hizo con Postman

Ejercicios

Ejercicios Propuestos de Desarrollo API con Spring

  • En los ejercicios propuestos se desarrollaron funcionalidades prácticas para afianzar el uso de Laravel, como mostrar la echa actual al acceder a la raíz del sitio, añadir validaciones y mensajes de error en el formulario de registro de estudiantes, implementar operaciones CRUD completas con visualización en una tabla responsive usando Tailwind CSS, y finalmente refactorizar el proyecto hacia una API REST utilizando controladores tipo resource y pruebas de endpoints con Postman.

PHP y Laravel

Conclusión

La práctica mostró cómo unir Apache, PHP y Laravel para levantar rápidamente un backend robusto y elegante. El uso de migraciones versionadas y Eloquent simplificó la interacción con MySQL, mientras que Tailwind  CSS permitió crear una UI limpia sin esfuerzo adicional.

Reflexión

Laravel ofrece velocidad de desarrollo y una sintaxis expresiva, pero exige conocer su filosofía (Service Container, Facades, Providers) para evitar “magia” opaca. Dominar Composer y la configuración inicial de PHP es clave para entornos productivos. Próximos pasos sugeridos: autenticación Jetstream/Breeze, despliegue en un VPS y pruebas automatizadas con Pest.
